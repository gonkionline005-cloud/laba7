1. Какие jQuery-функции использовались?
   
В рамках этого проекта для реализации интерактивных элементов использовался ряд ключевых функций библиотеки jQuery. Прежде всего, это $(document).ready(), которая гарантирует, что наш JavaScript-код будет выполнен только после полной загрузки HTML-страницы.
Для поиска элементов в структуре страницы используются селекторы ($()), позволяющие легко обращаться к элементам по их классам или ID, например, $('.tab-btn') или $('#draggable').
Обработка пользовательских действий реализована через метод .on('click', ...) для всех интерактивных кнопок. Переключение вкладок (Task A) основано на методах .addClass() и .removeClass() для управления видимостью контента через CSS-класс active.
Для реализации плавной прокрутки (Task B) используется метод .animate() с параметром scrollTop. И, наконец, для реализации сложного функционала (Task D) были задействованы плагины jQuery UI: метод .draggable() для перетаскивания блока и .datepicker() для создания интерактивного календаря.   

2. Что показалось быстрее/проще — jQuery или чистый JS?

Сравнивая два подхода в данном проекте, можно сделать следующий вывод:
jQuery оказался быстрее и проще для реализации основных DOM-манипуляций (например, переключение вкладок) за счет своего лаконичного синтаксиса и краткости кода. Кроме того, использование плагинов jQuery UI сделало реализацию сложного функционала, такого как Drag & Drop, невероятно быстрой и простой.
Чистый JS (Vanilla JS) в данном случае оказался надежнее для AJAX-запросов. Хотя его синтаксис (fetch с async/await) сложнее, чем у устаревшего $.get, он является современным стандартом и, в отличие от оригинальной jQuery-секции, успешно загрузил цитату, показав высокую надежность.
Таким образом, для большинства повседневных задач jQuery экономит время, но для современных веб-стандартов и надежности Vanilla JS предпочтительнее.

3. Что из этого вы бы оставили, если писали современный сайт?
   
Для написания современного сайта я бы выбрал гибридный подход с явным приоритетом на Чистый JavaScript.
Чистый JS (Vanilla JS): Я бы использовал его для всех основных задач: AJAX-запросов (через fetch), DOM-манипуляций (переключение вкладок) и даже плавной прокрутки (scrollIntoView({ behavior: 'smooth' })). Современный Vanilla JS очень быстр, не требует загрузки дополнительной библиотеки и полностью заменяет большую часть функций jQuery.
jQuery UI: Я бы оставил использование jQuery и jQuery UI только в тех случаях, когда требуется быстро интегрировать сложные и уникальные виджеты (как Drag & Drop или Datepicker), для которых не существует простого нативного решения или современной, легкой замены. Это позволяет сэкономить время, не разрабатывая сложный функционал с нуля.
